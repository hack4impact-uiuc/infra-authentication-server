# Verify
| Endpoint | /verify [POST] |
| ------ | ----------- |
| Header   | token |
| Response 200 | success message, role |
| Response 400 | error message |

(1) If the request doesn't have a header with a token, it will return an error message "Malformed request, doesn't have a token"

(2) Verifies the JWT token, and otherwise returns the error message "JWT Malformed"

(3) Verifies the user exists in the database, and if not returns the error message "User does not exist in the DB."

(4) Responds with a success message "User Verified", along with their role
